# RFC 2119 Keywords

The key words "MUST", "MUST NOT", "REQUIRED", "SHALL", "SHALL NOT",
"SHOULD", "SHOULD NOT", "RECOMMENDED", "MAY", and "OPTIONAL" in all
documents in this repository are to be interpreted as described in
[RFC 2119](https://www.ietf.org/rfc/rfc2119.txt).

| Keyword | Meaning |
|---------|---------|
| **MUST** / **REQUIRED** / **SHALL** | Absolute requirement. Violation is a defect. |
| **MUST NOT** / **SHALL NOT** | Absolute prohibition. |
| **SHOULD** / **RECOMMENDED** | May be ignored with good reason, but implications must be understood. |
| **SHOULD NOT** / **NOT RECOMMENDED** | May be done with good reason, but implications must be understood. |
| **MAY** / **OPTIONAL** | Truly optional. Implementations that omit it must still interoperate. |
