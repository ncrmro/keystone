# Create Task Note

## Objective

Create a structured permanent note in `notes/` to track an agent task. The note captures the
task goal, links it to VCS artifacts, and becomes the living record for the work session.

## Task

1. **Collect context**: Gather the task title, project slug, and any known artifact references
   (issue number, milestone, repository). Ask the operator if any are missing and required.

2. **Create the note**:

   ```bash
   zk new notes/ --title "<task title>" --no-input --print-path
   ```

   Save the returned path as the note path for subsequent steps.

3. **Write frontmatter**: Open the note and set the required frontmatter fields:

   ```yaml
   ---
   id: "<generated by zk>"
   title: "<task title>"
   type: permanent
   created: <ISO 8601 timestamp>
   author: <unix username of agent, e.g. agent-drago>
   tags:
     - project/<slug>
     - status/in-progress
   ---
   ```

   - `project/<slug>` MUST be present at creation time.
   - `status/in-progress` MUST be the initial status tag.
   - Add `repo:gh:<owner>/<repo>` if a repository is known.
   - Add `issue:gh:<owner>/<repo>#<number>` if an issue is known.
   - Add `milestone:gh:<owner>/<repo>#<number>` if a milestone is known.

4. **Write body**: Add the required body sections:

   ```markdown
   ## Objective

   <One or two sentences summarising the task goal.>

   ## Progress

   - [ ] (update as work proceeds)
   ```

5. **Commit the note**:

   ```bash
   git add notes/
   git commit -m "chore(notes): create task note — <task title>"
   ```

6. **Verify**: Run `zk list notes/ --tag "project/<slug>" --format json | head` and confirm the
   new note appears.

## Output Format

Write `.deepwork/tmp/task_note_path.md`:

```markdown
# Task Note Created

- **Path**: notes/<id> <title-slug>.md
- **Title**: <task title>
- **Tags**: project/<slug>, status/in-progress [, repo:gh:..., issue:gh:..., ...]
- **Commit**: <git short hash>
```

## Important Notes

- If `zk new` fails (directory permissions, missing `.zk/` directory), STOP and report the error.
  Direct the operator to run `notes/init` first if the notebook is not initialised.
- Do NOT proceed to `enrich_tags` if the note was not committed successfully.
- The note path output from this step is the required input for `enrich_tags` and
  `comment_shared_surface`.
