# Team Schedules

Keystone agent teams operate on a shared calendar hosted on Stalwart CalDAV. Recurring cadence events are created by the CPO agent (Luce) using `calendula` and are visible to any CalDAV client connected to the same Stalwart instance.

## Recurring cadences

| Event | Recurrence | Time | Purpose |
|-------|-----------|------|---------|
| `[Team] Weekly Retrospective` | Every Friday | 23:00 | Review what shipped, what was blocked, and what is next |
| `[Team] Portfolio Review` | Every Monday | 09:00 | Health assessment across all active projects via the `portfolio/review` workflow |

Events use the `[Team]` prefix per the [agentic calendar naming convention](#naming-convention).

## Naming convention

All agent-managed calendar events use a bracketed prefix to distinguish them from personal human events:

| Prefix | Meaning |
|--------|---------|
| `[Team]` | Shared cadence — all agents participate; human has read access |
| `[AgentName]` | Agent-specific event (e.g., `[Luce] Monthly Portfolio Review`) |

The human operator's personal events (no prefix) are read-only to agents.

## Tooling

Calendar management uses [`calendula`](https://github.com/pimalaya/calendula), a CalDAV CLI configured per agent via agenix secrets.

### List events

```bash
calendula events list default
```

### Create an event

Events are created by passing a raw iCalendar payload. Use `EDITOR` to inject the content non-interactively:

```bash
cat > /tmp/event.ics <<'EOF'
BEGIN:VCALENDAR
VERSION:2.0
PRODID:-//Keystone Agent//EN
BEGIN:VEVENT
UID:unique-event-id@keystone
SUMMARY:[Team] Weekly Retrospective
DTSTART:20260403T230000
DTEND:20260404T000000
RRULE:FREQ=WEEKLY;BYDAY=FR
DESCRIPTION:Weekly team retrospective.
STATUS:CONFIRMED
END:VEVENT
END:VCALENDAR
EOF

EDITOR="cp /tmp/event.ics" calendula items create default
```

### Delete an event

```bash
# Get the event ID from the list
calendula events list default

# Delete by ID
calendula items delete default <event-id>
```

## Agent configuration

The `calendula` config lives at `~/.config/calendula/config.toml` and is generated by the Keystone NixOS module. It inherits credentials from the agent's mail account via agenix.

```toml
[accounts.luce]
default = true

caldav.home-uri = "https://mail.example.com/dav/cal/agent-luce"
caldav.auth.basic.username = "agent-luce"
caldav.auth.basic.password.command = ["sh", "-c", "tr -d '\n' < /run/agenix/agent-luce-mail-password"]
```

## DeepWork workflow integration

The `[Team] Portfolio Review` event on Monday is intended to trigger the `portfolio/review` DeepWork workflow automatically once scheduler CalDAV integration is complete (tracked in [#199](https://github.com/ncrmro/keystone/issues/199)).

The `executive_assistant/manage_calendar` DeepWork workflow is used to manage team calendar events interactively.

## Cross-calendar visibility

Once Stalwart ACLs are configured (tracked in [#199](https://github.com/ncrmro/keystone/issues/199) Phase 3), agents and the human operator will be able to read each other's calendars. Until then, events created by agents are only visible on the agent's own CalDAV account.
